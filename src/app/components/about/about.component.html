About Project Thaqalayn

Objectives

  1. Proliferation of ahadith of ahlulbayt online
  2. Act as a data source: making hadith datasets open source and available to download for others to build on top of
  3. Act as a reference: every hadith should have a unique link that can be shared and will bring others to the same hadith
  4. Act as an aggregation portal for all relevant data that becomes available

Technical Design Principles

There are a few principles that the site is developed on:

  • All static data used to show a single page should be captured in a single data (json) file
  • Any static data should be reproducible from scratch - No ad-hoc edits to data files, everything should be scripted

History

Ideas of creating such a site on Quran and ahadith had been on my mind for at least a decade,
and in this time I've tried several times to produce something around this concept but to no avail.

In recent years Ive come to realise the shortage of free English ahadith resources online.

COVID-19 provided an unprecedented opportunity of extra time and energy to be allocated to this project
and alhamdullellah the project was kicked off.

Inspirations

There are many things that shaped the project to get it to be where it is now, here are some:

Tanzil.net

There is not enough good things I can say about tanzil.net I think they've done an excellent job over the years to be a great resource for Quran text on the net.

The thing that most inspires me about this site is that they provide Quran text for download,
which is obviously a great resouce that provides a foundation for developers like myself to build upon.

Making the data behind the site freely accessible to all

Sunnah.com

MashaAllah our Sunni brothers are way ahead in propagating their ahadith and making them more accessible.
I was impressed by the simplicity and accessibility of the design and based the look and feel of the project on this site to get started.

One of the barriers of building a site for me has always been the design,
so having seen a suitable template certainly removed the barrier for me to start.

Creating an easy navigable site for reading ahadith

Not being able to find a free host for my API

So when I started out the backend of the site was actually a REST API.
I started using by using the free tier services by Heroku, which is really good btw.
But I quickly learnt that I couldn't find a free hosted database service online and the free ones came with very limited restrictions.
So this led me to re-evaluate the architecture of the site, and the result was serving static
json files which is pretty unconventional, but it works and I can run it for free so I'm happy.
Plus I get to have all the data version controlled, which comes with other benefits
like being able to have a review process for accepting changes.

Keeping it simple and serving static content
